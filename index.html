What is Fujihack?

  • Fujihack a research project aiming to reverse-engineer and improve Fujifilm cameras.
  • The main project goals are to achieve interoperability and tweak firmware functionality.

What's Been Done?

  • Code execution through patched firmware
  • Portable debugger patch with brute-force RAM searching & patching
  • Patch RAM code to enable instant read/write/exec abilities over USB
  • Complete RAM and flash dump (1GB, 67mb)
  • Extend 14 minute record limit
  • Downgrade lenses stuck on buggy official firmware
  • Read IO: buttons, gyroscope, accelerometer
  • Fix commonly broken buttons with button remapping
  • Documented hidden scripting abilities
  • Draw debug text and rasterizer objects
  • Interface with embedded internal SQLite
  • Graphics rendering onto OpenGL buffer
  • Background task creation, deletion, semaphores, task hooks, events
  • Primitive card speed tests (capped at 35mbps)
  • Gryoscope controlled Tetris, 7fps DOOM port
  • POSIX compliant compatibility layer

Hacked Models

  • X-A2 @ 1.30 (Bin loader, USB code runner)
  • XF1 @ 1.01 (USB code runner)
  • Firmware downgrade has been tested on some lenses. (same firmware format used)

What's being worked on?

  • Researching less intrusive methods for code execution
  • Creating a custom menu with apps, scripts, and real time patching and tweaking
  • Bypassing error codes and remapping buttons to 'repair' broken or flawed cameras
  • Reverse-engineering and tweaking imaging and video code

Would it work with my camera?

Not all Fuji cameras are related to this project. See the supported models list.

What is it written in?

The hack is written in C and ARM assembly. Fuji cameras typically run an ARM11-based SoC. Newer cameras are based on Cortex-A7

When will you work on X?

Fujihack has and will always be non-commercial, hobbyist, and therefore will have no plans or deadlines. If you desperately need a feature on your camera, hacking it should be a last resort in most cases.
